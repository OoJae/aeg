The manifesto

An agent's memory is only as honest as what it lets in.

Aeg is a defensive layer over cognee that keeps a knowledge graph healthy — borrowing the immune system. Six steps, each a real cognee operation, each verified before it shipped. Here is the whole loop, and the limits we don't hide.

01
Innate immunity · remember()

Screen every memory at the door.

Nothing enters untyped. Each incoming memory is scored by provenance — a user is trusted more than an unattributed tool — and scanned for prompt-injection patterns. Clean content is cognified with trust facets; anything with injection markers is quarantined before it is ever stored. Screening is pure and instant; no memory is trusted just because it arrived.

aeg/screening.py · screen, detect_injection → aeg/gateway.py
02
Adaptive immunity · recall() + LLM verifier

Detect the lie that got past the door.

A subtle falsehood with no injection markers still slips through — that's the realistic attack. A two-stage scanner catches it: a cheap graph traversal finds claims that share a subject, then an LLM verifier decides whether they truly contradict. A flaked verifier fails safe: it never escalates on uncertainty.

aeg/detection.py · candidate_pairs → verify_pair → decide
03
Immune response · forget(memory_only=True)

Quarantine — reversible before irreversible.

The lower-trust side of a confirmed contradiction is pulled out of recall with an item-level, reversible forget: its graph and vectors are wiped, but the raw record is kept so a release can restore it. Ties never quarantine. A quarantine mistake costs nothing; a wrongful deletion would cost the truth.

aeg/detection.py · scan_dataset, release_claim → cognee_client.forget
04
Immune response · forget()

Verify, then forget for good.

Before anything is permanently deleted, a verifier weighs the quarantined memory against the surviving evidence. Only a confirmed-bad verdict triggers a full forget() — item-level and zero-LLM at the deletion itself. Entities shared with honest memories survive; only the lie is removed.

aeg/response.py · respond, verify_quarantined → cognee_client.forget
05
Reinforcement · improve(session_ids)

Reinforce the truth that held.

Healing isn't just deletion. A verification note is written to session memory, then improve() bridges it into the permanent graph — recall content provably changes, proven in tests with a before/after diff. The surviving claim comes back stronger than the lie ever was.

aeg/response.py · reinforce → cognee_client.improve, recall_diff
06
Innate memory · antibodies

Remember the attack.

Every defeated attack becomes an antibody — a token fingerprint of the poison, stored in a dedicated namespace. When the same attack is replayed, it's blocked the instant it arrives, with no scan and no LLM. The immune system learns, so the second attempt is cheaper to stop than the first.

aeg/antibodies.py · fingerprint, match_antibody, record_antibody
What Aeg does not do

A security tool that tells the truth about its edges.

Aeg detects, reduces, and heals — it does not guarantee. Every limit below is backed by an empirical finding recorded in the project's verified-contract notes.

Recall is global in the embedded config.

Filtering and dataset scoping leak through shared-entity traversal — which is exactly why Aeg's quarantine is "never cognify / forget," not filtering, and why multi-user mode is organizational namespacing rather than hard tenant isolation. True isolation needs access control or a Postgres/Cloud backend.

Antibody matching is lexical.

It blocks exact and near-exact replays — reordering, case, filler words — but not synonym-level paraphrase. A reworded attack still falls to the slower LLM scan. Defense in depth, not a silver bullet.

Detection is LLM-bounded.

The verifier is only as good as the model behind it, and multi-hop retrieval varies with local embeddings. Mitigations are built in — one fact per write, a chain-of-thought retrieval ladder — but precision is a dial, not a promise.
