An immune system for machine memory

Memory that heals.

Aeg is an immune system for an AI agent's memory, built on cognee. It screens what goes in, detects the lies, and forgets them — then reinforces the truth. Poison it. Watch it recover.

How Aeg uses cognee

Four operations, all load-bearing.

Not decoration. Each of cognee's memory operations does real work in the loop — and every behavior was empirically verified before a line of feature code.

01
remember()

Screened, typed ingest

Innate immunity scores every incoming memory by provenance and scans it for injection. Clean content is cognified with trust facets; blocked content is never written at all.

aeg/gateway.py · screening.screen → cognee_client.remember
02
recall()

Answers, and evidence

The agent's Q&A runs on graph-traversal lanes — a two-hop question the pure-vector lane can't answer. The same op proves the heal: recall before and after, side by side.

aeg/gateway.py · cognee_client.recall · recall_diff
03
forget()

The immune response

Two-tier and item-level. Reversible quarantine wipes a memory from recall but keeps the raw record; permanent removal deletes it after a verification gate. Zero LLM calls.

aeg/detection.py · aeg/response.py · cognee_client.forget
04
improve()

Reinforce the truth

A verification note is written to session memory, then bridged into the permanent graph — recall content provably changes. The truth comes back stronger than the lie ever was.

aeg/response.py · reinforce → cognee_client.improve
